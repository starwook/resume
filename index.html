🎈 Introduce

사용자의 관점에서 생각합니다.

코드는 사용자에게 가치를 제공하기 위한 수단 중 하나라고 생각합니다.

사용자 중심의 접근을 하여 개발한 서비스가 배포 한 달만에 하루 최대 5,000회 이상의 API 요청을 처리한 경험이 있습니다.

유지보수성 높은 코드에 대해 고민합니다.

확장성과 재사용성이 높은 설계와 가독성 높은 코드 작성을 추구합니다.

리팩토링에 투자하는 시간을 아까워 하지 않습니다.

자원과 결과를 함께 고려합니다.

문제 해결 이후에도 끊임없이 더 나은 방법을 모색합니다.

하지만 주어진 자원도 고려하며 그 안에서 최선의 결과를 내려고 합니다.

  • Java 17, Spring-Boot 3.2.2, MySql, Jmeter, Pinpoint, AWS
  • 목표
    • 1vCpu 2GB의 WAS, DB 서버에서 선착순 이벤트 페이지를 구현하고 이 과정에서 최소한의 리소스로 최대한의 효율을 내라는 미션을 받았습니다.
  • 성과
    • 전공 지식을 활용해 실제 개발에서 발생한 동시성 문제를 해결하였습니다.
    • 팀과 함께 나아갈 방향을 고민하고 실천했습니다. 문서화를 통한 정보 공유로 팀원이 동일한 문제를 신속히 해결할 수 있었고, 프론트엔드에 신뢰성 있는 테스트 기반 API 명세서를 제공해 협업 효율을 높였습니다.
  • 수행 업무
    1. 트래픽이 몰릴 수 있는 선착순 이벤트 기능 구현
      • Mysql 비관적 락을 통한 선착순 이벤트 당첨자 수 동시성 제어
      • DB 커넥션을 얻는 과정에서 발생하는 병목 현상 개선
        • 로컬 캐시를 사용하여 DB 접근 최소화
        • DB 접근을 위한 thread-safe한 대기열을 구현하여 DB 접근 최소화
          • 1초에 200건의 동시성 제어가 필요한 요청에서 평균 시간 7초 → 2.5초, 최대 시간 14초 → 4.5초로 단축
        • 개별 요청별 DB 조회 대신, CompletableFuture로 요청을 큐에 적재하고 주기적 배치 처리로 DB 접근 최소화
          • 1초에 2000건의 동시성 제어가 필요한 요청에서 평균 시간 50초 → 0.6초, 최대시간 79초 → 2초로 단축
        • DB 커넥션 풀을 분리하여 병목 현상 범위 최소화
        • 요청당 한 개의 DB 커넥션을 사용하도록 하여 데드락 및 커넥션 누수 방지
      • Jmeter로 스트레스 테스트를 진행
        • 유저 플로우를 반영하여 스트레스 테스트 시나리오를 작성
    1. 테스트 작성
      • 10개 내외의 API에서 50개 이상의 통합 테스트, 단위테스트 작성
      • Spring Rest Docs를 통해 테스트를 기반으로 한 API 명세를 작성하여 협업 효율 향상
    1. 협업
      • 프로젝트 관리 도구로 Jira 사용
        • 백로그 작성, 주 단위 스프린트, 스크럼 회의 등을 통하여 애자일 방법론 도입
      • 매일 경험하고 부딪힌 일을 문서화
        • 트러블 슈팅 해결 과정, 기술 버전 및 설치 커맨드들을 문서화하여 팀원간의 동일한 문제를 겪었을 때 빠르게 해결
